# E2E Runs UX Seed File

## Manual Verification Seed
Follow these steps to manually verify the E2E runs UX flow. Document results in [docs/TASK_VERIFICATION_TEMPLATE.md](docs/TASK_VERIFICATION_TEMPLATE.md).

## Prerequisites
- [ ] Backend server running (`scripts/server.sh` or `uvicorn app.main:app`)
- [ ] Frontend dev server running (`npm run dev`)
- [ ] Database (`backend/app.db`) initialized with at least one historical or active run record
- [ ] WebSocket connections enabled and accessible at `/ws/runs/{id}` and `/ws/events`

### Verification Steps

1. **Run List Rendering**
   - Load the main application UI.
   - Verify that the Run History/List panel displays correctly.
   - Confirm that `RunCard` components render the run title, status, and timestamp.
   - Check badge colors and labels match `runStatusBadgeClass` logic in `frontend/src/types/runs.ts`.

2. **WebSockets & Real-Time Updates**
   - Trigger an action in the UI that initiates a run or event broadcast.
   - Verify that `useWebSocket` hook captures events without dropping messages.
   - Observe real-time status transitions in the UI (e.g., pending -> running -> completed).

3. **Run Detail & Log Viewer**
   - Click on a run ID to open `RunDetailDrawer`.
   - Verify `RunLogPanel` and `TerminalPanel` render correctly.
   - Confirm that `/nothink` toggle and model selection reflect correctly in the UI state.
   - Check that tool calls and thought blocks render inside `ToolCallCard` and `MessageBubble`.

4. **Agent & Artifact Flow**
   - Verify that `AgentPanel` and `ReviewArtifactPanel` load without crashing.
   - Confirm that `ApproveDialog` triggers as expected on agent tool calls.
   - Test patch diff rendering via `coderPatchDiff` in the UI.

5. **Error & Edge Cases**
   - Terminate the backend while a run is active; verify graceful fallback or connection lost toast via `toast.ts`.
   - Reload the page during an active WebSocket session; confirm state recovery or safe reconnection.

### Known Manual-Only Checks
- Database seeding requires manual or script-based DB population for consistent UX testing.
- Live environment deployment or cross-browser testing is out of scope for this seed.
